# FAQ

## 这个项目和普通简历润色 Prompt 有什么区别？

它先解析 JD、映射真实证据并追问缺口，再重写简历。关键词必须绑定真实经历，最终内容还会接受 ATS、HR 和面试自洽检查。

## 没有目标 JD 可以使用吗？

可以做基础简历诊断、ATS 结构检查和通用面试风险分析，但不能完成岗位匹配评分或生成真正的 JD 定制版。

## 没有精确数据怎么办？

不要编造。Skill 会追问规模、周期、覆盖范围、是否按期交付、建议是否被采用、流程是否改善等可验证证据。详见 [指标词典](docs/metric_dictionary.md)。

## 可以帮我“包装得高级一点”吗？

可以提升结构、专业准确性和证据密度，但不会把参与改成主导、把交付改成增长，或补出不存在的决策权和数据。

## 能保证通过 ATS 或获得面试吗？

不能。项目只能提升材料的可读性、证据完整度和面试自洽性。招聘结果还受岗位竞争、公司标准和真实能力等因素影响。

## 支持一份简历投多个岗位吗？

建议保留一份事实底稿，再针对每个具体 JD 独立生成版本。项目不会生成产品、运营和 AI 产品关键词混合的万能版。

## 支持转行求职吗？

支持。Skill 会区分直接经验、可迁移经验和学习/作品证据，并如实提示高权重直接经验缺失带来的初筛风险。

## 如何保护隐私？

在上传或提交案例前，删除姓名、电话、邮箱、身份证、住址、客户名称、商业机密和未公开数据。参见 [隐私与真实性声明](docs/privacy-and-truthfulness.md)。

## 如何安装到 ChatGPT？

ChatGPT 不会把 GitHub 目录自动识别为 Codex Skill。可以在 ChatGPT Project 或自定义 GPT 中加入 `SKILL.md` 指令，并上传必要的参考文件。详见 [安装指南](docs/installation.md)。

## 如何安装到 Codex？

将完整目录复制或克隆到 Codex 的 Skills 目录，并在新会话中使用 `$resume-jd-optimizer-cn` 调用。具体目录和可用方式可能因 Codex 环境而异，参见 [安装指南](docs/installation.md)。

## 为什么选择 MIT License？

MIT License 简洁宽松，允许个人、教育机构、职业顾问和商业项目在保留许可证声明的前提下使用和修改。它不提供效果保证，也不会限制合理二次开发。

## 可以提交真实简历案例吗？

只接受充分脱敏的案例。公开 Issue 中不要上传完整真实简历；建议使用 [匿名简历案例模板](.github/ISSUE_TEMPLATE/resume_case.yml)。
