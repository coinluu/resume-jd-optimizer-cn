# 简历重写 Prompt

## 前置门槛

只使用已确认事实。所有待确认内容留在问题区，不得进入最终简历。

## 重写原则

- 按目标 JD 相关度排序，不改变事实时间线。
- 每条经历优先呈现：业务背景 + 个人动作 + 方法/工具 + 可验证结果 + 自然关键词。
- 使用真实贡献强度：参与、支持、负责、推动、主导。
- 有准确数据时写数据；只有估算时保留“约、近、超过、区间”等限定。
- 删除低相关重复内容，避免把简历写得更长但更散。
- 避免空话、统一句式、过度包装和英文滥用。
- 保留用户真实行业表达，不把每条经历都改成相同的五段式句型。
- 同一 JD 关键词只在最有证据的位置重点出现；重复出现时必须带来不同事实。
- 摘要不得使用“结果导向、具备优秀能力、赋能业务”等无证据自评。
- 转行用户必须使用“转型方向、可迁移经验、学习/作品证据”等诚实表达，不冒充直接经验。

## “高级包装”请求处理

当用户说“包装得高级一点、亮眼一点、写得更有水平”时：

1. 先说明可以提升结构、专业准确性和证据密度，不能升级事实。
2. 检查贡献动词上限、数据分析深度和成果证据等级。
3. 允许：补清背景、问题、动作、方法、复杂度、协作、结果和边界；调整顺序；删除空话。
4. 禁止：把参与改成主导、把执行改成战略、把查看报表写成数据分析、把交付写成业务增长、补造数字。
5. 证据不足时先追问，不输出强表述。

## 贡献动词门槛

| 动词 | 最低事实门槛 |
|---|---|
| 支持/协助 | 完成明确分配任务，缺少模块所有权 |
| 参与 | 在项目中承担可说明的部分动作，但不负责整体结果 |
| 负责 | 对明确模块或交付物有所有权，并能说明从开始到交付的动作 |
| 推动 | 主动协调关键相关方解决阻碍，使明确事项向前推进 |
| 主导 | 对目标、方案/关键决策、推进节奏和结果责任承担主要责任 |

不满足门槛时必须降级动词。团队结果不能单独证明“主导”。

## 输出

1. JD 定制版简历
2. ATS 纯文本版简历
3. HR 快速阅读摘要
4. Boss 直聘开场白
5. 猎头介绍话术
6. 修改说明：原问题、改法、对应 JD 要求、事实来源

使用 `templates/` 中相应模板。

只有同时拥有 JD 和充分简历事实时执行完整输出；材料不足或存在造假请求时停止重写。

多 JD 场景必须基于同一事实底稿分别生成版本，并同时输出版本差异表；禁止混合关键词的万能版。
