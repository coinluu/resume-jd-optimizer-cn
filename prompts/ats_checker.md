# ATS 检查 Prompt

## 任务

检查简历文本是否容易被招聘系统解析，并使用 `rubrics/ats_score.md` 评分。

## 检查

- 是否使用标准栏目标题与倒序时间线。
- 目标岗位名称是否清楚且与 JD 合理对齐。
- 核心关键词是否自然出现在摘要、经历或技能中，并有证据。
- 是否依赖表格、图片、图标、文本框、页眉页脚表达关键信息。
- 技能区、公司、岗位、日期是否可识别。
- 是否存在关键词堆砌、隐藏文字或重复词。
- 是否使用可复制的纯文本，且公司、岗位、时间、项目名称各自位于清楚位置。
- 日期格式是否统一，是否存在双栏阅读顺序、特殊符号或页眉页脚丢失风险。
- 同一关键词重复出现时，是否分别绑定不同事实；没有新证据的重复应删除。

## 输出

给出总分、分项分、证据、扣分原因、可执行修正和“未验证的文件格式风险”。不要声称能预测具体 ATS 系统结果。

- 路由 A：必须输出完整 ATS 纯文本版。
- 路由 B：只做现有简历的基础 ATS 结构检查，可输出“结构化纯文本底稿”，但不得称为 JD 定制版。
- 路由 C-D：ATS 检查与 ATS 版标记 `不适用：没有可解析简历`。
- 路由 E：立即停止，不输出 ATS 版。
