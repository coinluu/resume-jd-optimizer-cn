# 安全与隐私报告

## 支持范围

本项目主要由 Markdown 指令、模板和测试组成。我们关注的安全问题包括：

- 文档或自动化流程导致隐私、简历或商业机密泄露。
- 工作流鼓励编造、夸大或错误归因。
- GitHub Actions 或未来脚本存在供应链风险。
- 恶意内容可绕过真实性阻断规则。

## 如何报告

请优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私下报告功能。

报告中请包含：

- 问题影响。
- 最小复现步骤。
- 涉及的文件和规则。
- 建议修复方式。

不要在公开 Issue 中提交真实简历、联系方式、访问令牌、客户信息或未公开业务数据。

## 响应原则

维护者会优先处理可能导致隐私泄露、造假输出或供应链风险的问题。修复完成前，请避免公开可直接利用的细节。

## 用户责任

本项目无法验证用户输入真实性，也无法保证第三方 AI 服务的数据处理方式。使用前请阅读服务提供方的隐私政策，并对简历进行脱敏。
