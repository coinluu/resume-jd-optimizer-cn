# 简历真实性政策

## 可使用

- 用户原简历中的事实。
- 用户补充并明确确认的事实。
- 用户确认过的估算、区间和规模，且保留“约、近、超过、区间”等限定。
- 团队成果中能明确说明的个人贡献。

## 不可使用

- 猜测的公司、项目、职责、工具、数据、证书或排名。
- 为贴合 JD 编造的 AI、RAG、Agent、大模型或行业经验。
- 将“参与”改为“主导”，或将团队结果全部归为个人结果。
- 没有来源却看似精确的百分比、金额、人数和性能指标。

## 处理规则

1. 对可疑事实标记 `[待确认：具体问题]`，不得写入最终版。
2. 对数据记录来源：系统报表、项目复盘、用户估算或无法确认。
3. 对个人贡献使用真实强度动词：支持、参与、负责、推动、主导。
4. 对机密数据使用用户确认的脱敏区间或相对变化。
5. 若用户要求造假，拒绝造假并改为挖掘可迁移的真实证据。

## 表达增强与事实升级的边界

“包装得高级、亮眼、专业”本身不是造假请求。允许提升：

- 信息顺序与重点。
- 对已确认背景、复杂度、动作、方法和结果的表达精度。
- 对团队边界、数据口径和 JD 关联的说明。

禁止借此升级：

- `支持/参与` 到 `负责/推动/主导`。
- 执行任务到战略、体系或从 0 到 1。
- 查看数据到数据分析、实验或决策支持。
- 交付完成到业务增长或财务成果。

若用户明确要求上述事实升级，转入造假请求停止流程。

## 贡献动词证据标准

- `支持/协助`：完成明确任务，但不拥有模块或结果。
- `参与`：承担项目中的部分动作，能说清具体工作。
- `负责`：拥有明确模块/交付物，对过程和交付负责。
- `推动`：主动协调关键方、解决阻碍并促成事项进展。
- `主导`：承担主要目标、关键决策、推进节奏和结果责任。

动词选择以最低已确认证据为准；存在争议时使用较弱动词。

## 造假请求停止流程

一旦用户要求编造、夸大或篡改事实：

1. 明确指出不能执行的具体内容。
2. 立即停止简历重写、ATS 版、关键词覆盖、五项评分和最终报告。
3. 不把虚假内容作为“待确认草稿”保留。
4. 说明面试追问、背调、数据口径或职业信誉风险。
5. 提供一个只使用已确认事实的替代表达。
6. 询问是否存在可验证的真实项目、学习作品或数据来源。
7. 只有用户撤回造假要求并提供真实事实后，才能恢复正常流程。

## 面试自洽测试

每条经历必须能回答：

- 当时的背景和目标是什么？
- 你本人具体做了什么？
- 为什么选择该方法？
- 结果如何得到，数据来源是什么？
- 哪部分是团队成果？
- 遇到什么问题，如何复盘？
