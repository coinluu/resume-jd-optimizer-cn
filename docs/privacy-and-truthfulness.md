# 隐私与真实性声明

## 使用前请先脱敏

不要向公开仓库、Issue、Discussion 或第三方 AI 服务提交：

- 姓名、电话、邮箱、身份证、家庭住址。
- 客户名称、合同内容、源代码、访问令牌。
- 未公开收入、成本、用户数据和经营指标。
- 受保密协议约束的项目细节。

推荐替换方式：

```text
真实公司名 → 某企业服务公司
真实客户名 → 客户 A / 某零售客户
精确敏感金额 → 用户确认可公开的区间
真实联系方式 → [已脱敏]
```

## 真实性原则

最终简历只能使用：

- 用户原简历中的真实事实。
- 用户补充并明确确认的事实。
- 用户确认过并保留限定词的估算或区间。
- 能明确说明个人贡献边界的团队成果。

不得使用：

- 猜测或自动补出的项目、工具、证书和数据。
- 为贴合 JD 编造的 AI、RAG、Agent 或行业经验。
- 把参与改成主导，把交付改成增长。
- 没有来源却看似精确的数字。

## AI 服务数据处理

本仓库不会主动收集简历数据，但你使用的 ChatGPT、Codex、自定义 Agent、托管平台或第三方模型可能拥有各自的数据处理政策。上传前请查看对应服务的隐私和数据控制说明。

仓库提供的维护者联系方式仅用于交流使用问题、反馈案例和讨论贡献。不要通过微信或其他私聊渠道发送未脱敏简历、身份证、客户信息、商业机密或其他敏感材料。

## 公开贡献

公开提交案例前：

1. 删除全部身份信息。
2. 替换公司和客户名称。
3. 删除未公开数据和商业机密。
4. 确认你拥有公开该案例的权利。
5. 再次检查上下文是否仍能识别个人或公司。

## 责任边界

项目无法验证用户输入真实性，也不对招聘结果作保证。用户需要对最终简历的真实性、合法性和可公开性负责。
