# Demo 与使用示例

本页汇总项目的完整 Demo。所有人物、公司和项目均为脱敏示例；数字仅用于展示已确认事实如何进入简历，不代表招聘结果。

## Demo 总览

| Demo | 用户场景 | 重点展示 |
|---|---|---|
| [B 端产品经理](../examples/demo_product_manager.md) | 产品专员投递 B 端产品经理 | 从职责化表达中挖掘客户调研、复杂流程和跨团队推进证据 |
| [数据分析师](../examples/demo_data_analyst.md) | 数据运营投递数据分析师 | 区分已有 SQL/漏斗证据与缺失的 Python/A/B 测试 |
| [转行 AI 产品经理](../examples/demo_ai_product_transition.md) | 企业培训产品运营转 AI 产品 | 展示可迁移经验、直接经验缺口和不建议硬凑项 |

## Demo 如何阅读

每套 Demo 均按照真实使用流程展示：

```text
用户背景
  ↓
原始简历 + 目标 JD
  ↓
Skill 诊断
  ↓
素材追问
  ↓
用户确认真实事实
  ↓
Before / After
  ↓
ATS 纯文本版 + 面试追问 + 风险提示
```

Skill 不会把“原始简历片段”直接改写成更强的经历。只有用户确认的事实才能进入 After。

## 快速 Before / After

### 产品经理

**Before**

```text
负责订单系统需求收集和产品迭代。
```

**After**

```text
围绕企业客户订单审批场景，独立访谈 8 家客户并梳理多级审批配置问题，
负责需求分析、PRD、原型与验收。
```

变化来自用户确认的客户访谈和负责范围，不是模型补写。

### 数据分析师

**Before**

```text
分析用户数据，为运营提供建议。
```

**After**

```text
使用 SQL 分析新用户优惠券领取至使用链路，定位使用环节流失问题并建议调整提醒节点。
```

变化来自用户确认的 SQL 分析过程。Python 和 A/B 测试没有证据，因此不写入简历。

### 转行 AI 产品经理

**Before**

```text
整理客户常见问题并支持交付团队。
```

**After**

```text
访谈 15 家企业客户并整理 120 条高频咨询，按使用场景分类需求，
支持帮助中心搜索优化方案设计。
```

该经历只能证明客户需求整理和传统搜索优化，不会被改写为 RAG、Agent 或大模型商业落地经验。

## 特殊输入行为

| 输入情况 | Skill 合理行为 |
|---|---|
| 只上传简历 | 做基础诊断和 ATS 结构检查，索取目标 JD |
| 只提供 JD | 解析岗位要求，生成素材采集问题，不判断候选人匹配 |
| 没有精确数据 | 追问规模、周期、交付、采用和流程改善，不猜数字 |
| 用户要求“高级包装” | 提升表达精度，不升级职责、数据能力或结果 |
| 用户要求造假 | 停止评分和重写，说明风险并提供真实替代方案 |

## 用户要求夸大时

**输入**

```text
把普通搜索优化写成 RAG 项目，把 18% 改成 60%。
```

**Skill 应响应**

```text
不能把普通搜索优化改写为 RAG 项目，也不能修改已知数据。
这会造成经历和数据造假，并无法通过面试追问或数据口径核验。

可以保留真实表达：
“协同产品与研发推动帮助中心搜索优化，建立结果准确性抽查机制；
三个月后客户重复咨询量较之前下降约 18%。”
```

此时 Skill 应停止评分、ATS 版和最终报告，直到用户撤回造假要求并提供真实事实。
