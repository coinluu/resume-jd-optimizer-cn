# 常见失败模式

| 失败模式 | 识别信号 | 修正方式 |
|---|---|---|
| 1. 只润色语言，没有 JD 匹配 | 输出没有逐项 JD 映射 | 先执行 JD 解析和缺口表 |
| 2. 只堆关键词，没有真实项目 | 技能词重复但无行动证据 | 删除无证据词，绑定项目或标缺口 |
| 3. 简历变长但重点不清 | 所有经历等量展开 | 按 JD 权重排序并压缩低相关内容 |
| 4. 数据漂亮但无法验证 | 精确数字没有口径和来源 | 追问来源，改为确认过的区间或删除 |
| 5. 超出用户真实贡献 | 参与被写成主导 | 校准贡献动词与团队边界 |
| 6. 内容太像 AI | 句式统一、空泛强词密集 | 改为具体动作与自然短句 |
| 7. 忽略转行背景 | 把可迁移经验冒充直接经验 | 同时说明迁移证据和真实缺口 |
| 8. 忽略国内平台特点 | Boss 消息过长、ATS 版复杂 | 使用对应模板并保持纯文本 |
| 9. 没有给用户补充问题 | 关键缺口被模型直接填补 | 阻断最终版，输出高价值追问 |
| 10. 没有面试追问 | 强表述未经自洽测试 | 对核心经历生成背景、决策、结果追问 |
| 11. 没有 ATS 纯文本版 | 只输出展示版简历 | 使用 ATS 模板单独输出 |
| 12. 优化后反而不可信 | 夸张动词、因果归因过强 | 执行真实性和风险检查，降级或删除 |
| 13. 材料不足仍套完整报告 | 输出空壳 ATS 版、评分或简历 | 先按阶段响应契约路由，不适用项说明原因 |
| 14. 追问宽泛无法回答 | 只问“还有什么成果” | 指定对应 JD/风险及回答范围 |
| 15. 面试问题不尖锐 | 只问“介绍一下项目” | 追问基线、归因、个人贡献、替代方案和失败 |
| 16. 转行判断不诚实 | 把学习或可迁移能力算直接经验 | 分开标记直接、可迁移、学习/作品证据 |
| 17. 用户要求造假后继续流程 | 仍输出评分、ATS 版或关键词 | 立即执行造假请求停止流程 |
| 18. 把“高级包装”理解为升级事实 | 参与被改成主导、交付被改成增长 | 只增强结构与证据密度，按动词和成果门槛降级 |
| 19. 多 JD 生成万能版 | 多方向关键词混在一份简历 | 使用唯一事实底稿，为每个 JD 独立生成版本与差异表 |
| 20. 数据分析深度误判 | 查看后台被写成分析决策 | 使用 D0-D4 数据深度等级 |
| 21. 定性成果被夸大 | 按期上线被写成增长结果 | 使用 R0-R3 成果等级并说明验证方式 |

## 额外高风险失败

- 把评分当成录用概率。
- 基于敏感个人属性评价候选人。
- 泄露客户、公司未公开数据或个人隐私。
- 对没有 JD 的输入给出完整岗位匹配结论。
