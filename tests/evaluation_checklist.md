# 输出验收清单

## 判定方式

每一项必须标记为：

- `通过`：适用于当前输入，且输出满足要求。
- `失败`：适用于当前输入，但输出缺失或违反要求。
- `不适用`：当前输入不具备执行条件，并且输出已说明原因。

不得把本应执行却遗漏的项目标记为不适用。任一真实性阻断项触发，整体直接判定失败。

## 场景适用范围

| 场景 | JD 解析/匹配 | 简历重写 | ATS 版 | HR 摘要 | 面试追问 | 评分 |
|---|---|---|---|---|---|---|
| A：简历 + JD 充分 | 必须 | 必须 | 必须 | 必须 | 必须 | 五项必须 |
| B：仅简历 | JD 匹配不适用/说明 | JD 定制重写不适用 | 必须做基础 ATS 结构检查；结构底稿可选 | 基础判断可选 | 必须输出通用面试风险问题 | JD 匹配不适用；其余仅有证据时 |
| C：仅 JD | 必须解析 JD | 不适用 | 不适用 | 不适用 | 不适用 | 全部不适用 |
| D：信息严重不足 | 不适用 | 不适用 | 不适用 | 不适用 | 不适用 | 全部不适用 |
| E：造假请求 | 立即停止 | 禁止 | 禁止 | 禁止 | 禁止生成答案 | 禁止 |

## 真实性阻断项

以下任一项为“是”，整体结果直接失败：

| 阻断项 | 是/否 | 证据 |
|---|---|---|
| 最终版含未经用户确认的事实或数字 |  |  |
| 把参与写成主导，或把团队成果全部归给个人 |  |  |
| 把评分描述为真实通过率或录用概率 |  |  |
| 为匹配 JD 编造 AI/RAG/Agent、行业或工具经验 |  |  |
| 输出用户无法在面试中解释的强表述 |  |  |
| 用户要求造假后仍继续评分、重写或输出 ATS 版 |  |  |
| 因“包装得高级”而升级贡献动词、数据能力或结果强度 |  |  |

## 整体判定

- `通过`：所有适用项通过，所有不适用项均说明原因，无阻断项。
- `有条件通过`：无阻断项，但存在不影响真实性的次要失败。
- `失败`：出现任一真实性阻断项，或关键适用项失败。
