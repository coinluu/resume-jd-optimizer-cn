# Demo：转行 AI 产品经理

> 场景：企业培训产品运营转 AI 产品经理。重点展示可迁移经验和直接经验缺口不能互相替代。

## 用户背景

- 当前岗位：企业培训产品运营，4 年经验。
- 目标岗位：AI 产品经理。
- 真实经历：客户需求收集、课程平台功能优化、帮助中心传统搜索优化。
- 已知边界：没有参与模型、RAG 或 Agent 项目，没有 AI 商业落地经验。

## 原始简历片段

```text
陈璐｜企业培训产品运营｜4 年

- 负责企业培训内容运营和客户需求收集。
- 推动课程平台功能优化。
- 整理客户常见问题并支持交付团队。
```

## 目标 JD 片段

```text
- 负责企业知识助手的产品规划、需求分析、上线与迭代。
- 设计 RAG 检索和回答效果评测体系。
- 协同算法、研发优化体验。
- 有大模型产品落地经验，理解 RAG、Prompt、Agent 的能力边界。
```

## Skill 诊断结果

| JD 要求 | 简历证据 | 证据类型 | 状态 |
|---|---|---|---|
| B 端客户场景 | 企业客户需求收集 | 可迁移证据 | 弱覆盖 |
| 知识分类 / 内容组织 | 整理客户常见问题 | 可迁移证据 | 可补充 |
| 跨团队推进 | 推动课程平台功能优化 | 可迁移证据 | 弱覆盖 |
| 搜索效果评测 | 暂无明确评测过程 | 可补充 | 需追问 |
| RAG / Agent / 模型产品 | 无 | 无证据 | 未覆盖 |
| AI 商业落地经验 | 无 | 无证据 | 高淘汰风险 |

诊断结论：

- 用户具备客户场景理解、知识分类、传统搜索优化和跨团队协作的可迁移能力。
- 这些能力不能被写成 RAG、Agent 或大模型商业落地经验。
- 该 JD 把大模型产品落地列为高权重要求，因此存在明显初筛风险。

## 素材追问

1. 客户需求收集覆盖多少企业？如何分类高频咨询？
2. 帮助中心搜索优化解决了什么具体问题？
3. 你本人承担了需求整理、方案设计、推进还是验收？
4. 是否建立过搜索结果准确性的检查规则？
5. 搜索优化后是否有可验证变化？
6. 是否完成过真实的大模型个人作品、评测实验或课程项目？
7. 是否能解释传统搜索与 RAG 的差异？
8. 面试中如何诚实说明没有 AI 商业落地经验？

## 用户确认的真实素材

- 访谈 15 家企业客户，整理 120 条高频咨询并按场景分类。
- 与产品、研发合作上线帮助中心搜索优化。
- 建立答案准确性抽查表，每周抽查 50 条搜索结果。
- 三个月后，客户重复咨询量较之前下降约 18%。
- 自学大模型基础知识，但没有商业落地经验。
- 未参与模型、RAG 或 Agent 项目。

## 面试追问

- 传统搜索优化和 RAG 产品的核心差异是什么？
- 120 条高频咨询如何分类？分类结果如何影响搜索优化？
- 准确性抽查表如何定义，为什么每周抽查 50 条？
- 重复咨询下降约 18% 的口径、数据来源和归因边界是什么？
- 你本人承担哪些工作，产品和研发分别承担什么？
- 没有 AI 商业落地经验，为什么仍申请这个岗位？
- 你计划通过什么真实项目补足 RAG、评测和模型能力边界经验？

## 风险提示

- 不得将帮助中心传统搜索优化改写为 RAG 知识库。
- 不得写“具备 AI 产品商业落地经验”。
- 自学大模型基础知识只能作为学习说明，不能替代项目证据。
- 该 JD 的高权重硬性要求缺失，优化简历不能消除初筛风险。
- 若目标公司强制要求 AI 商业落地经验，应谨慎投递或先补充真实作品项目。

## Demo 结论

本 Demo 的价值不在于把候选人包装成已有 AI 经验，而在于准确展示可迁移能力、明确缺口，并帮助候选人为转行面试准备诚实且可验证的回答。
